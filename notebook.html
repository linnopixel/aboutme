DNS tuh apaan, sih?

Intinya, Domain Name System (DNS) itu kayak buku telepon buat internet. Kerjanya ngubah nama domain yang gampang diingat, kayak `google.com`, jadi alamat IP yang isinya angka-angka (`142.250.190.46`), biar komputer kita tahu ke mana harus nyambung. Jadi, lo gak perlu repot-repot hafalin IP address-nya deh!

Langkah-Langkah: Instalasi & Konfigurasi DNS Server (bind9) di Debian

Install bind9, Simpel Banget!

Pertama, pastiin dulu sistem lo udah di-update, terus langsung aja install paket bind9-nya pakai perintah ini:

sudo apt update
sudo apt install bind9

Konfigurasi bind9, Anti Pusing!

Udah beres instalasi? Mantap! Sekarang kita lanjut setting file named.conf.local buat nentuin zona forward sama reverse-nya. Jangan lupa, ganti contoh.com pake domain lo sendiri dan 192.168.1 pake subnet IP lo, ya.

// Konfigurasi zona forward (nama ke IP)
zone "contoh.com" {
    type master;
    file "/etc/bind/db.contoh.com";
};

// Konfigurasi zona reverse (IP ke nama)
zone "1.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192";
};

Habis itu, bikin file zona forward db.contoh.com. Caranya gampang kok, tinggal copy file template yang udah ada:

sudo cp /etc/bind/db.local /etc/bind/db.contoh.com
sudo nano /etc/bind/db.contoh.com

Isi file db.contoh.com-nya kayak gini:

$TTL 604800
@ IN SOA localhost. root.localhost. (
    2023010101 ; Serial
    604800     ; Refresh
    86400      ; Retry
    2419200    ; Expire
    604800 )   ; Negative Cache TTL
;
@   IN NS   localhost.
@   IN A    192.168.1.100
www IN A    192.168.1.100
mail IN A   192.168.1.101

Terakhir, bikin file zona reverse db.192. Ikutin aja langkah ini:

sudo cp /etc/bind/db.127 /etc/bind/db.192
sudo nano /etc/bind/db.192

Isi file db.192:

$TTL 604800
@ IN SOA localhost. root.localhost. (
    2023010101 ; Serial
    604800     ; Refresh
    86400      ; Retry
    2419200    ; Expire
    604800 )   ; Negative Cache TTL
;
@   IN NS   localhost.
100 IN PTR contoh.com.
100 IN PTR www.contoh.com.
101 IN PTR mail.contoh.com.

Tambah Domain ke Web Server, Gampang Banget!

Biar domain lo bisa diakses, lo perlu setting virtual host di web server (misalnya Apache atau Nginx). Ini contohnya buat Apache:

sudo nano /etc/apache2/sites-available/contoh.com.conf

Isi file konfigurasi Apache:


    ServerAdmin webmaster@localhost
    ServerName contoh.com
    ServerAlias www.contoh.com
    DocumentRoot /var/www/contoh.com
    ErrorLog ${APACHE_LOG_DIR}/error.log
    CustomLog ${APACHE_LOG_DIR}/access.log combined

Aktifin virtual host-nya, terus restart service-nya:

sudo a2ensite contoh.com.conf
sudo systemctl restart apache2

Testing di Client (Linux), Kuy!

Sekarang, edit file /etc/resolv.conf di client lo biar ngarah ke server DNS yang barusan lo bikin:

sudo nano /etc/resolv.conf

Tambahin baris ini (ganti IP-nya pake IP server DNS lo):

nameserver 192.168.1.100

Abis itu, lo bisa ngetes DNS server-nya udah jalan apa belom pake perintah nslookup atau dig. Kalo keluar info IP-nya, berarti udah berhasil!

nslookup www.contoh.com
dig contoh.com

Mikrotik, Nih!

Setting DHCP, Pake Terminal Aja!

Buat nyalain DHCP Server di Mikrotik, lo bisa pake perintah setup singkat ini di terminal:

/ip dhcp-server setup

Ikutin aja petunjuknya, masukin interface, network, gateway, sama DNS server. Atau, lo bisa juga manual gini:

/ip pool add name=dhcp-pool ranges=192.168.88.10-192.168.88.254
/ip dhcp-server add name=dhcp-server interface=bridge-local address-pool=dhcp-pool disabled=no
/ip dhcp-server network add address=192.168.88.0/24 gateway=192.168.88.1 dns-server=8.8.8.8

Rules Firewall, Biar Aman!

Firewall itu gunanya buat ngatur traffic yang masuk-keluar jaringan lo. Ini contoh rule dasar buat ngeblok akses ke IP tertentu dari jaringan lokal lo.

/ip firewall filter add chain=forward src-address=192.168.88.150 dst-address=10.0.0.10 protocol=tcp action=drop comment="Blokir akses ke 10.0.0.10"

HTML Basics, Gampang!

HTML (HyperText Markup Language) itu bahasa dasar buat bikin halaman web. Setiap web pasti punya struktur dasar yang sama. 

<!DOCTYPE html>
<html lang="id">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Judul Halaman</title>
</head>
<body>
    <h1>Halo, Dunia!</h1>
    <p>Ini adalah paragraf pertama saya.</p>
</body>
</html>

CSS Level Up!

Kalau udah jago dasar-dasarnya, saatnya naik level pake Flexbox dan Grid. Flexbox itu mantap buat ngatur tata letak satu dimensi (baris atau kolom), kalo Grid cocok buat tata letak dua dimensi (baris dan kolom).

Nih, contoh pake Flexbox:

.container {
    display: flex;
    justify-content: center; /* Menempatkan item di tengah secara horizontal */
    align-items: center;     /* Menempatkan item di tengah secara vertikal */
}

IoT ESP32, Mainin Aja!

Setup Arduino IDE, Sat Set!

Biar bisa ngoding ESP32 pake Arduino IDE, lo harus tambahin URL board manager-nya dulu. Buka File > Preferences terus tambahin URL ini di kolom Additional Boards Manager URLs:

https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json

Abis itu, buka Tools > Board > Boards Manager, cari "esp32", terus install deh.

Project Smart Home: Nyalain-Matiin LED Pake Web Server!

Nih, kode dasarnya buat nyalain dan matiin LED yang nyambung ke pin GPIO 23 pake web server sederhana di ESP32. 

#include <WiFi.h>
#include <WebServer.h>

const char* ssid = "your_SSID";
const char* password = "your_PASSWORD";

WebServer server(80);
const int ledPin = 23;

void handleOn() {
  digitalWrite(ledPin, HIGH);
  server.send(200, "text/plain", "LED ON");
}

void handleOff() {
  digitalWrite(ledPin, LOW);
  server.send(200, "text/plain", "LED OFF");
}

void setup() {
  pinMode(ledPin, OUTPUT);
  Serial.begin(115200);
  WiFi.begin(ssid, password);
  while (WiFi.status() != WL_CONNECTED) {
    delay(1000);
    Serial.println("Connecting to WiFi...");
  }
  Serial.println("Connected to WiFi!");
  Serial.println(WiFi.localIP());

  server.on("/on", handleOn);
  server.on("/off", handleOff);
  server.begin();
}

void loop() {
  server.handleClient();
}
